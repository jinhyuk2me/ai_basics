# 작업 가이드
- 모든 답변과 문서 작성은 한국어로 진행합니다.
- 이 저장소는 Obsidian Vault이므로, 문서 내용에서 `# 제목` 형태의 최상위 헤더는 작성하지 않습니다. 파일명과 Obsidian 헤더가 중복되지 않도록 `##`부터 시작하세요.
- 관련 있는 노트는 `[[노트명]]` 링크를 사용해 상호 연결하고, 노트 말미에 "관련 노트" 섹션을 배치해 다음 학습 경로를 안내합니다.
- 이모지는 절대 사용하지 않습니다. 모든 문서 작성 시 이모지를 배제합니다.
- 수식은 MathJax/LaTeX 표기법을 사용합니다. 인라인은 `$...$`, 블록 수식은 `$$...$$`로 감싸고, 예시는 아래와 같습니다.
  - 인라인: $e^{2i\pi} = 1$
  - 블록:
    $$
    \begin{vmatrix}a & b\\
    c & d
    \end{vmatrix}=ad-bc
    $$
  - 추가 문법은 *MathJax basic tutorial and quick reference* 및 *The TeX/LaTeX Extension List*를 참고하세요.

## PyTorch Tensor 노트 템플릿
- 모든 PyTorch Tensor 계열 노트는 아래 섹션을 기본으로 유지합니다.
  1. `## TL;DR`
  2. `## 언제 쓰나`
  3. `## 주요 API` (필요 시 하위 테이블로 분리)
  4. `## 실습 예제`
  5. `## 실수 주의`
  6. `## 관련 노트`
- 표는 동일한 형식(헤더 포함, 굵은 글씨 최소화)으로 작성하고, 불필요한 이모지나 장식은 사용하지 않습니다.
- 코드 블록은 바로 실행 가능한 예제로 구성하고 `python` 언어 표시를 붙입니다.

## 편집 원칙
- 기존 노트 구조를 해치지 않되, 개념·예제·주의사항을 보강해 학습 흐름을 살립니다.
- 내용을 보강할 때는 중복을 최소화하고, 이미 다룬 주제는 해당 노트로 링크를 연결합니다.
- 새로운 노트를 추가할 경우에도 위 템플릿을 따르며, 허브 노트(`PyTorch Tensor.md`)의 로드맵을 업데이트합니다.
